## Key Design Decisions

| Decision | Rationale |
|---|---|
| Clean Architecture layering | Domain has zero framework deps; swap DB/HTTP without touching business rules |
| Webhook has no OAuth2 | External payment processor cannot provide Bearer token; validated by payment_code lookup |
| No Catalog call on "paid" webhook | Vehicle already marked "sold" at sale initiation; double-update would cause 409 |
| Best-effort Catalog restore on "cancelled" | Sale status is the source of truth; Catalog failure doesn't block the response |
| Lazy DB engine init | Avoids KeyError on DATABASE_URL during test collection |
| vehicle_price_at_sale snapshot | Listing reflects sale-time price, not current Catalog price |
